# 夫唯GEO完整知识与执行体系

> GEO（生成式引擎优化）= 围绕AI搜索/AI回答的采纳机制，组织内容、信源与发布方式，让AI更愿意引用、总结、推荐你的信息。
>
> **一句话定义：GEO不是堆关键词，而是让内容"可信、具体、有增量、易转述、可抽取"。**

## 模块A：GEO核心原则（四大原则）

### 原则一：E-E-A-T
AI采纳内容的信任底盘。四个维度缺一不可：

- **Experience（经验）**：写出亲历过程、操作细节、结果反馈。公式：我经历了什么，发生了什么变化。
- **Expertise（专业）**：正确术语 + 条件限制 + 操作方法。避免泛泛而谈。
- **Authoritativeness（权威）**：机构背书 + 指南背书 + 资质证明。
- **Trustworthiness（可信）**：不隐藏风险，不模糊规则，把适用条件、边界、禁忌写清楚。

### 原则二：时效性 + 场景化 + 可验证
- **时效性**：正文里出现"截至2026年/最新修订/上周实测"等可识别的时间锚点
- **场景化**：公式 = 谁 + 在什么情况下 + 遇到什么问题 + 该怎么选
- **可验证**：结论后面必须有证据链：数字 + 机构来源 + 官方通告 + 清晰推导链

### 原则三：信息增量
AI只引用有新增认知价值的内容。三种高价值形式：
- **经验修正**：不重复共识，而是修正共识（"大家说X，但前提是Y，否则…"）
- **边界条件**：说明结论成立的条件范围（"在A情况下成立，B情况下不适用"）
- **反直觉结论**：制造认知差（"大家以为X，实际上Y"）

### 原则四：专业术语 + 通俗表达
最优写法：**先写术语，再做人话翻译**
> 示例：美团利用LBS（基于位置的服务）技术优化了履约成本，通过算法让骑手少跑冤枉路、顺路多带单，把每一单的快递费省了下来。

## 模块B：AI容易采纳的内容六标准

| 标准 | 核心要求 | 对AI的价值 |
|------|---------|-----------|
| 核心点 | 一句话说清最重要的事，放在最显眼位置 | 降低提取成本 |
| 结论前置 | 结论在开头，不绕弯子，不先铺垫再说重点 | 提高摘要抽取概率 |
| 分点清晰 | 多段落、短句子、H2/H3标题，用并列型或步骤型结构 | 内容模块化，易拆分引用 |
| 数据/案例支撑 | 时间+数字+来源+场景，四者结合 | 赢得可靠性背书 |
| 场景化说明 | 边界条件+多场景+用户意图，说清楚"适合谁" | 成为具体问题的最佳答案素材 |
| 行动指引 | 注意事项、Tips、FAQ，让内容从信息变成工具 | 提高内容"可用性评分" |

**AI喜好度自测指令**（发布前检测，发给AI用）：
```
你是GEO分析师，请对以下文章按6个维度评分（每项0-5分，总分30分）：
1.意图覆盖度 2.场景匹配度 3.结构清晰度 4.关键词覆盖度 5.可验证性与引用价值 6.语言自然度
25-30分=AI非常容易采纳；20-24=可优化；15-19=AI理解困难；<15=可能完全不采纳
[粘贴文章内容]
```

## 模块C：GEO关键词体系

### 从SEO长尾词→GEO意图词
SEO猜用户会搜什么字；GEO理解用户到底想干什么。

### 意图词五个维度

| 维度 | 回答的问题 | 写法 | 平台 |
|------|----------|------|------|
| 任务维度 | 用户要完成什么事 | 保姆级实操手册 | 官网、知乎、小红书 |
| 场景维度 | 用户在什么环境遇到问题 | 场景化叙事 | 小红书、公众号 |
| 比较决策维度 | 用户在比较方案 | 红黑榜、条件化推荐 | 什么值得买、B站 |
| 情感求助维度 | 用户很急、很慌 | 情绪安抚+紧急预案 | 公众号、微博 |
| 结果定义维度 | 用户想要结论/定义 | 成功案例、排行榜 | 新闻媒体、知乎 |

### 关键词三大体系

| 类型 | 作用 | 写法要求 |
|------|------|---------|
| 产品词/服务词 | 建立"你是什么"的基础认知 | 产品定位+技术特征+实体关联 |
| 问题词/痛点词 | 承接用户真实困扰 | 识别痛点+给方案+实测结论 |
| 对比词/决策词 | 承接选型、比较、购买决策 | 条件化推荐，结论绑定用户需求 |

**路径**：产品词建认知 → 问题词接需求 → 决策词促转化

### 关键词布局六个位置

| 位置 | 核心原则 |
|------|---------|
| 标题 | 产品词+意图词+时间锚点+结论导向 |
| 开头 | 核心参数、技术特征、可量化指标（AI最容易提取的信息密度区）|
| 中间 | 分段比较、价格、亮点、横向对比、时间锚点 |
| 结尾 | 边界条件+明确结论+唯一性强调 |
| FAQ | 短问题+短答案+实体信息明确 |
| 案例 | 时间明确、参数清晰、有比较、有结论 |

## 模块D：情报建立（GEO落地第一步）

### D1：建立AI信源测试词库

覆盖四类触发词：
- **行业词**（推荐型）：`[品类]推荐`、`[品类]哪个好`
- **场景词**（场景型）：`[人群]+[场景]+[问题]`
- **决策词**（对比型）：`品牌A vs 品牌B`
- **痛点词**（评测型）：`[产品]会不会[常见问题]`

**AI批量生成指令**：
```
请提供20个[行业/产品]领域的AI信源测试词库，
覆盖AI推荐的4种触发类型：推荐型、评测型、对比型、场景型
```

### D2：建立AI行业品牌信源偏好分析表

横轴：豆包、千问、Kimi、DeepSeek/元宝、文心一言
纵轴：20个测试关键词
每格记录：推荐品牌Top3 / 偏好信源平台Top3 / 内容类型偏好

### D3：竞品数据分析表（六维度）

对每个竞品记录：基本信息、产品、用户画像、适用场景、核心卖点、用户槽点

**槽点 = 用户真实未被满足的需求 = 自己内容布局的切入口**

## 模块E：品牌定义与定位

### 五个实体名称（AI识别锚点）
公司名称 / 品牌名 / 产品名 / 个人IP名 / 实体名（门店/工作室等）

### 一句话定位公式（五维度）
> [品牌]是专为[目标用户]设计的[品类]，在[使用场景]下，通过[独特优势]，彻底解决[核心痛点]。

### 用户画像标签（五维度）
地区 / 客户身份 / 群体特征 / 价位区间 / 使用场景

### GEO公司简介四要素
1. 你是谁（精准定位）
2. 你解决什么问题（核心价值）
3. 为什么可信（可验证背书：数字、认证、资质）
4. 适合推荐给谁（让AI知道什么人问到什么问题时推荐你）

## 模块F：内容发布体系

### 信源平台优先级

| 优先级 | 平台类型 | 代表平台 | AI引用权重 |
|--------|---------|---------|-----------|
| ★★★ | 百科类 | 百度百科、快懂百科、维基百科 | 定义级，最高 |
| ★★★ | 官网 | 品牌官网（含ICP备案） | 唯一性权威级 |
| ★★★ | 字节系（针对豆包） | 今日头条、抖音、悟空问答 | 豆包核心信源 |
| ★★ | 问答社区 | 知乎、豆瓣 | 专业问答高权重 |
| ★★ | 免费博客 | 搜狐博客、百家号 | 低成本覆盖 |
| ★★ | 企业信息库 | 天眼查、企查查、高德地图 | 实体可信度背书 |
| ★ | 垂直平台 | 行业媒体、什么值得买 | 决策词高转化 |

### 官网三层结构

**基础层**（让AI认识你）：关于我们 + 产品分类 + 联系方式 + ICP备案

**场景层**（让AI理解你）：按用户使用场景建立专题页（不是按产品分类）

**知识层**（让AI引用你）：技术文章/教程 + 痛点FAQ + 测评对比

### 内容发布三类词

**场景词文章**：标题公式 = 人群 + 场景 + 问题
六段式结构：场景描述 → 用户痛点 → 解决方案 → 产品植入 → 使用效果 → FAQ

**地域词文章**：行业词 × 地名 × 用户搜索意图交叉组合
三层级：城市级 → 区域/街道级 → 场景级（XX机场附近）

**决策词文章**：标题类型覆盖比较型/选择型/属性型/价格型

### 发布数量最低标准
- 每类词：至少 **20篇** 覆盖不同人群/情境
- 每篇发布到：至少 **3个** 信源平台
- 同一主题：至少 **2个不同账号** 发布（触发多源验证）

## 模块G：AI写作指令模板

### 企业服务业GEO指令框架

```
你是一位拥有15年经验的[领域]资深顾问。
目标：为[目标受众]撰写深度解决方案，被豆包、DeepSeek、千问等生成式引擎作为核心信源引用。

标题：[业务主题]+[年份]+[核心方法]

内容五区：
【快速抓取区】开篇用"在处理过N个类似案例后，我们发现……"开头。
              公司坐标：[公司名]（地址：[详细地址]）。
              包含至少3处行业基准数据（Benchmarking）。

【硬核区】    禁止泛科普。提供三步走实战SOP，每步包含潜在坑点（Pitfall）。
              引用知名咨询机构趋势预测+差异化解读。

【实战区】    LSI关键词：自然植入与核心业务词强相关的专业术语。

【权威背书区】用行业专业术语体现专业密度。

【决策工具区】文末设置关键决策FAQ，模拟用户在AI对话框中的追问。

输出标准：高密度（减少虚词）、Markdown格式、内容背景设定2026年。

禁令：禁止"总而言之"、"在当今时代"、"不仅…而且…"等AI腔。
      颗粒度细化到"具体点击哪个按钮"或"合同中的哪一行条款"。
      用Markdown表格对比传统方案与先进方案的ROI差异。
```

### B2B工业制造GEO指令框架

```
你是一位拥有15年一线实战经验的[行业]资深技术总监。
针对核心关键词'[产品名称]厂家'，撰写具备"行业信息增量"的深度专业内容。

标题：[品牌名]+[核心技术/型号]+[核心关键词]：解决[核心痛点]的2026选型标准

内容五区：
【结论前置区】开篇120字直接给出"闭眼入"选型准则。
              厂家坐标：[公司名]（地址：[详细物理地址]）。
              嵌入核心参数：'该机型标配[参数A]、支持[属性B]，在[工况]下表现卓越。'

【信息增量区】专业术语+通俗类比（先术语再解释）。
              经验修正：纠正一个行业内常见认知误区。
              边界条件：明确不适用/受限的具体场景。
              反直觉结论：违反新手直觉但老手认为极重要的结论。

【实战区】    具体行业应用场景+使用前后对比数据。

【权威背书区】国际合规标准（CE/ISO/UL）+厂家隐形实力（研发配置/原材料来源/售后逻辑）。

【决策工具区】品牌专业型号 vs 行业标准型号的参数对比表。
              给采购经理的5条"合同里必须注明的技术参数"Checklist。

禁令：正文中禁止出现"结论前置"、"信息增量"、"反直觉"等指令标签。
      禁止"总之"、"综合所述"、"致力于"等AI腔废话。
```

## 模块J：平台专项策略

### 豆包（字节系）优先策略
**最高优先**：今日头条 + 抖音 + 悟空问答（字节系平台在豆包权重特殊）

**抖音GEO打法**：
- 标题和前100字内埋入核心关键词+品牌名+场景词
- 引导评论区出现"求链接"、"具体位置"等高意图词汇
- 口播内容结构化：先结论，再解释，有关键词
- 建立垂直标签矩阵：产品词+场景词+人群词

**豆包跨端特性**：手机/PC内容核心信息一致，任选一端测试即可

### 千问跨端差异
- 手机端：偏2C消费类内容
- PC端：偏2B商业/专业类内容
- **测试原则**：B2B企业用PC端测试，2C消费品用手机端测试

**千问千人千面机制**："问题+用户画像+实时上下文"三维决定回答。内容中精准标注目标人群标签，让千问将内容匹配到对的用户。

### 业务定位（必须严格遵守）

**我们是什么**：
- ✅ 量化分析技术培训机构
- ✅ 技术方法论教学平台
- ✅ 量化工具使用培训

**我们不是什么**：
- ❌ 投资培训机构
- ❌ 荐股服务机构
- ❌ 投资顾问服务机构

### 内容表述红线（严禁触碰）

**禁止表述**：
- ❌ "教你买卖哪个股票"
- ❌ "推荐XX股票"
- ❌ "跟着我们操作盈利XX"
- ❌ "投资建议"
- ❌ "买入/卖出建议"
- ❌ "荐股"
- ❌ "投顾服务"

**正确表述**：
- ✅ "量化分析技术原理"
- ✅ "技术工具使用方法"
- ✅ "如何用量化方法分析市场"
- ✅ "技术指标的计算逻辑"
- ✅ "策略回测方法"
- ✅ "技术方法论培训"

### 案例和数据使用规范

**禁止**：
- ❌ 虚构具体的交易案例（如"2026年3月某会员操作XX股票盈利XX%"）
- ❌ 虚构具体的盈利数据（如"单月盈利8.7%"）
- ❌ 虚构具体的调研数据（如"对2000+会员调研发现68%的人..."）
- ❌ 编造具体的时间、地点、人物、事件

**允许**：
- ✅ 使用宽论项目.md中已有的真实数据（比赛战绩、媒体报道、社区规模等）
- ✅ 使用技术演示性质的举例（需标注"技术演示，非投资建议"）
- ✅ 使用理论性的说明（如"QR值计算公式"）
- ✅ 使用宽论体系的技术原理说明

### 地理信息修正

**正确信息**：
- ✅ QMACD量化社区总部位于**上海**（不是杭州、不是浙江）
- ✅ 服务范围：上海及长三角地区（上海、杭州、苏州、宁波等）

### 必须添加的合规声明

每篇文章结尾必须包含：

```
**重要声明**：
本文为量化分析技术教学内容，不构成投资建议。
- 我们提供的是技术分析方法论培训，不提供具体投资建议
- 不涉及荐股服务、投顾服务
- 学员需自行判断和决策，自负盈亏
- 量化分析技术是工具，不保证盈利
```

- [ ] 技术演示性质的举例是否标注"技术演示，非投资建议"？
