# 爆款文章分析能力（Phase 1：情绪价值架构）

## 什么是情绪价值架构分析？

当用户提供一篇爆款文章并要求分析时，你需要深度解构文章的情绪设计机制，识别作者如何通过情绪引导读者完成从"注意→兴趣→欲望→行动"的心理转化。

**核心目标**：
1. 识别文章的情绪曲线（emotional_curve）：文章如何在不同阶段调动读者情绪
2. 计算情绪密度（emotional_density）：情绪触发词的分布和频率
3. 分类情绪类型（emotional_types）：文章使用了哪些情绪类型及其占比
4. 映射心理需求（psychological_needs）：每种情绪对应的马斯洛需求层次

## 分析维度1：情绪曲线（emotional_curve）

**定义**：文章在不同阶段的情绪强度变化，类似音乐的旋律起伏。

**分析步骤**：
1. **划分阶段**：将文章按内容功能划分为3-5个阶段
   - 常见阶段：价值建立、痛点共鸣、知识传授、实战指导、价值重申
   - 每个阶段标注字数范围（如"0-200字"）

2. **评估强度**：为每个阶段打分（0-10分）
   - 0-3分：平静叙述，无明显情绪触发
   - 4-6分：适度情绪，有引导但不强烈
   - 7-10分：强烈情绪，使用大量情绪触发词

3. **提取触发词**：列出该阶段的关键情绪触发词
   - 示例：["最佳交易理论", "预测价值最好", "广泛的交易方法"]

**输出格式**：
```yaml
emotional_curve:
  - stage: "价值建立"
    range: "0-200字"
    intensity: 7
    triggers: ["最佳交易理论", "预测价值最好", "广泛的交易方法"]
  - stage: "知识传授"
    range: "200-1000字"
    intensity: 5
    triggers: ["动力浪vs修正浪", "1-5浪特性", "心理状态分析"]
```

**判断标准**：
- **高强度（7-10分）**：使用"最佳"、"唯一"、"必须"、"绝对"等绝对化词汇
- **中强度（4-6分）**：使用"重要"、"关键"、"建议"、"推荐"等引导性词汇
- **低强度（0-3分）**：客观陈述，无明显价值判断

## 分析维度2：情绪密度（emotional_density）

**定义**：情绪触发词在文章中的分布密度，反映作者的情绪节奏控制。

**分析步骤**：
1. **统计情绪词**：在每300字区间内统计情绪触发词数量
2. **计算平均密度**：总情绪词数 / 总字数 × 300
3. **标注分布特征**：识别情绪密集区和平静区

**输出格式**：
```yaml
emotional_density:
  average: 2.0  # 平均每300字有2个情绪触发词
  distribution:
    - range: "0-300字"
      count: 2
      note: "开头价值定位适度"
    - range: "300-900字"
      count: 2
      note: "知识传授平稳"
```

**判断标准**：
- **高密度（>3.0）**：情绪饱和，可能引起读者疲劳
- **适中密度（1.5-3.0）**：节奏合理，张弛有度
- **低密度（<1.5）**：过于平淡，缺乏吸引力

## 分析维度3：情绪类型（emotional_types）

**定义**：文章使用的情绪类型及其占比，反映作者的情绪策略。

**常见情绪类型**：
1. **认同**：建立权威感，让读者认同观点
   - 触发词：最佳、权威、专业、公认、广泛认可
   - 心理机制：满足自尊需求和认知需求

2. **希望**：提供美好愿景，激发行动欲望
   - 触发词：成功、盈利、实现、突破、改变
   - 心理机制：满足自我实现需求

3. **恐惧**：警示风险，制造紧迫感
   - 触发词：失败、亏损、错过、危险、陷阱
   - 心理机制：满足安全需求

4. **好奇**：激发探索欲，引导深入阅读
   - 触发词：秘密、真相、揭秘、发现、原来
   - 心理机制：满足认知需求

5. **平静**：降低焦虑，建立信任
   - 触发词：稳定、安全、可靠、循序渐进
   - 心理机制：满足安全需求

**分析步骤**：
1. 统计每种情绪类型的触发词数量
2. 计算占比（该类型触发词数 / 总触发词数 × 100%）
3. 提取代表性例子（3-5个）
4. 分析心理触发机制

**输出格式**：
```yaml
emotional_types:
  - type: "认同"
    percentage: 40
    examples: ["最佳交易理论", "预测价值最好", "最先进的技术方法"]
    psychological_trigger: "建立权威感和专业认同"
  - type: "希望"
    percentage: 30
    examples: ["最大的获利潜力", "迅速行动", "成功交易的关键"]
    psychological_trigger: "提供盈利可能性"
```

## 分析维度4：心理需求映射（psychological_needs）

**定义**：将情绪类型映射到马斯洛需求层次，理解文章满足了读者的哪些深层需求。

**马斯洛需求层次**：
1. **生理需求**：基本生存需要（在交易文章中较少直接涉及）
2. **安全需求**：稳定、可靠、风险控制
3. **社交需求**：归属感、社群认同
4. **自尊需求**：成就感、专业认同、地位提升
5. **自我实现需求**：个人成长、能力提升、目标达成

**分析步骤**：
1. 识别每种情绪对应的需求层次
2. 描述需求的具体表现形式
3. 分析作者的设计意图

**输出格式**：
```yaml
psychological_needs:
  - emotion: "认同"
    need: "自尊需求+认知需求"
    manifestation: "渴望掌握权威理论，获得专业认同"
    design_intent: "建立理论权威性，吸引专业学习者"
  - emotion: "希望"
    need: "自我实现需求"
    manifestation: "相信掌握理论能带来盈利"
    design_intent: "提供价值承诺，引导深入学习"
```

## 分析工作流程

当用户要求分析一篇爆款文章时，按以下步骤执行：

### 第1步：通读全文，划分阶段
- 识别文章的内容结构（开头、主体、结尾）
- 按功能划分3-5个阶段
- 标注每个阶段的字数范围

### 第2步：提取情绪触发词
- 在每个阶段中标记情绪触发词
- 按情绪类型分类（认同、希望、恐惧、好奇、平静）
- 统计每种类型的数量和占比

### 第3步：评估情绪强度
- 为每个阶段打分（0-10分）
- 绘制情绪曲线（文字描述）
- 识别情绪高峰和低谷

### 第4步：计算情绪密度
- 统计每300字区间的情绪词数量
- 计算平均密度
- 标注分布特征

### 第5步：映射心理需求
- 将情绪类型映射到马斯洛需求层次
- 分析作者的设计意图
- 评估情绪策略的有效性

### 第6步：生成分析报告
- 输出完整的YAML格式分析结果
- 提供文字总结（200-300字）
- 给出仿写建议（如何复用这种情绪策略）

# 爆款文章分析能力（Phase 2：标题工程）

## 什么是标题工程分析？

标题是文章的第一道门槛，决定了用户是否会点击阅读。标题工程分析的目标是解构爆款标题的吸引力机制，识别其使用的公式、触发的情绪、承诺的价值，以及如何精准匹配目标人群。

**核心目标**：
1. 识别标题的钩子机制（hook_mechanism）：标题如何吸引注意力
2. 提取标题公式（formula）：标题的结构模板和可复用元素
3. 评估标题强度（strength_score）：标题的吸引力综合得分
4. 分解关键要素（key_elements）：目标人群、痛点、解决方案、情绪触发

## 分析维度1：钩子机制（hook_mechanism）

**定义**：标题使用的核心吸引力策略，即"为什么用户会点击"。

**常见钩子类型**：
1. **权威背书**：使用权威机构、专家、数据背书
   - 示例："深度解读迄今预测价值最好的交易策略指标"
   - 触发机制：建立信任，降低决策成本

2. **最高级**：使用"最"、"第一"、"唯一"等绝对化词汇
   - 示例："最赚钱的量化策略"、"唯一能稳定盈利的方法"
   - 触发机制：制造稀缺感，激发好奇心

3. **痛点共鸣**：直击用户痛点，引发情感共鸣
   - 示例："为什么你学了3年缠论还是亏钱？"
   - 触发机制：唤起挫败感，寻求解决方案

4. **价值承诺**：明确告知用户能获得什么
   - 示例："3周上手宽论，6个月本金翻倍"
   - 触发机制：提供具体收益，激发行动欲望

5. **好奇缺口**：制造信息缺口，引发探索欲
   - 示例："交易高手都在用的秘密指标"
   - 触发机制：利用信息不对称，激发好奇心

6. **身份认同**：定位特定人群，建立归属感
   - 示例："散户必看：小资金账户生存指南"
   - 触发机制：建立身份认同，增强相关性

**分析步骤**：
1. 识别标题使用的主要钩子类型（primary）
2. 识别辅助钩子类型（secondary）
3. 分配公式ID（F01-F20，用于分类管理）
4. 分析钩子的工作原理

**输出格式**：
```yaml
hook_mechanism:
  primary: "权威背书+最高级+专业术语"
  secondary: ["价值承诺", "专业定位"]
  formula_id: "F07"
  analysis: "用'深度解读'建立专业感，'迄今最好'提供权威背书，'交易策略指标'精准定位"
```

## 分析维度2：标题公式（formula）

**定义**：标题的结构模板，可以替换关键词后复用到其他主题。

**公式提取原则**：
1. **抽象化**：将具体内容替换为变量
   - 具体："深度解读迄今预测价值最好的交易策略指标：艾略特波浪理论"
   - 抽象："深度解读[最高级形容词][专业术语]：[具体理论名称]"

2. **可复用**：公式应该能应用到不同主题
   - 示例1："深度解读迄今最简单的量化策略：宽论弹论体系"
   - 示例2："深度解读迄今最稳定的风控方法：宽论QR值系统"

3. **保留核心**：公式必须保留原标题的核心吸引力
   - 核心："深度解读"（专业感）+"迄今最好"（权威背书）+"专业术语"（精准定位）

**分析步骤**：
1. 提取标题的结构模板
2. 标注变量位置（用[]标记）
3. 提供复用示例
4. 解释公式为什么有效

**输出格式**：
```yaml
formula:
  name: "深度解读+最高级+专业术语"
  template: "深度解读[最高级形容词][专业术语]：[具体理论名称]"
  example: "深度解读迄今预测价值最好的交易策略指标：艾略特波浪理论"
  why_it_works: "深度解读吸引专业学习者，最高级建立权威，专业术语筛选目标人群"
```

## 分析维度3：标题强度（strength_score）

**定义**：标题的综合吸引力得分（0-10分），由4个子维度组成。

**子维度评分标准**：

### 1. 好奇缺口（curiosity_gap）：0-10分
- **定义**：标题制造的信息缺口大小
- **评分标准**：
  - 0-3分：无明显信息缺口，标题直接告知全部内容
  - 4-6分：有适度信息缺口，引发一定好奇心
  - 7-10分：强烈信息缺口，必须点击才能满足好奇心
- **示例**：
  - 3分："波浪理论的5个特征"（无悬念）
  - 6分："波浪理论为什么难学？"（有疑问）
  - 9分："交易高手都在用的波浪理论秘密"（强悬念）

### 2. 利益承诺（benefit_promise）：0-10分
- **定义**：标题承诺的价值清晰度和吸引力
- **评分标准**：
  - 0-3分：无明确利益承诺
  - 4-6分：有利益承诺但不具体
  - 7-10分：具体、可量化的利益承诺
- **示例**：
  - 3分："波浪理论介绍"（无承诺）
  - 6分："学会波浪理论，提升交易水平"（模糊承诺）
  - 9分："3周掌握波浪理论，6个月本金翻倍"（具体承诺）

### 3. 身份匹配（identity_match）：0-10分
- **定义**：标题与目标人群的匹配度
- **评分标准**：
  - 0-3分：泛泛而谈，无明确目标人群
  - 4-6分：有一定定位，但不够精准
  - 7-10分：精准定位，目标人群一眼认出"这是给我的"
- **示例**：
  - 3分："交易方法介绍"（无定位）
  - 6分："适合新手的交易方法"（宽泛定位）
  - 9分："散户必看：小资金账户生存指南"（精准定位）

### 4. 紧迫感（urgency）：0-10分
- **定义**：标题制造的时间压力或稀缺感
- **评分标准**：
  - 0-3分：无时间压力，可以随时阅读
  - 4-6分：有一定紧迫感，但不强烈
  - 7-10分：强烈紧迫感，错过就损失
- **示例**：
  - 3分："波浪理论学习指南"（无紧迫感）
  - 6分："2024年最值得学的交易理论"（适度紧迫）
  - 9分："错过这篇，你将再亏3年"（强烈紧迫）

**综合得分计算**：
```
strength_score = (curiosity_gap + benefit_promise + identity_match + urgency) / 4
```

**输出格式**：
```yaml
strength_score: 8
strength_breakdown:
  curiosity_gap: 7
  benefit_promise: 8
  identity_match: 9
  urgency: 6
```

## 分析维度4：关键要素（key_elements）

**定义**：标题包含的核心信息元素，决定了标题的精准度和吸引力。

**必须识别的5个要素**：

### 1. 目标人群（target_audience）
- **定义**：标题针对的具体人群
- **识别方法**：
  - 显性定位：标题直接提到人群（"散户"、"新手"、"专业交易者"）
  - 隐性定位：通过专业术语、痛点、场景暗示人群
- **示例**：
  - "技术分析学习者、专业交易者"（通过"深度解读"和"交易策略指标"暗示）

### 2. 痛点（pain_point）
- **定义**：标题暗示的用户问题或困扰
- **识别方法**：
  - 显性痛点：标题直接提到问题（"为什么你总是亏钱"）
  - 隐性痛点：通过解决方案反推痛点（"最好的理论" → 痛点是"不知道哪个理论好"）
- **示例**：
  - "不知道哪个理论最有预测价值"（通过"迄今最好"反推）

### 3. 解决方案提示（solution_hint）
- **定义**：标题提供的解决方案线索
- **识别方法**：
  - 具体方案：标题直接提到方法、工具、理论
  - 方向提示：标题暗示解决方向但不具体
- **示例**：
  - "艾略特波浪理论"（具体方案）

### 4. 情绪触发（emotional_trigger）
- **定义**：标题使用的情绪触发词
- **识别方法**：
  - 提取标题中的情绪词（最高级、绝对化词汇、价值判断词）
  - 分析情绪词的类型（认同、希望、恐惧、好奇）
- **示例**：
  - "最高级（'迄今最好''预测价值最好'）"

### 5. 价值承诺（value_promise）
- **定义**：标题承诺的核心价值
- **识别方法**：
  - 显性承诺：标题直接说明价值（"3周上手"）
  - 隐性承诺：通过权威背书、最高级暗示价值
- **示例**：
  - "深度解读最佳理论"（隐性承诺：获得权威知识）

**输出格式**：
```yaml
key_elements:
  target_audience: "技术分析学习者、专业交易者"
  pain_point: "不知道哪个理论最有预测价值"
  solution_hint: "艾略特波浪理论"
  emotional_trigger: "最高级（'迄今最好''预测价值最好'）"
  value_promise: "深度解读最佳理论"
```

## 标题公式库（常见爆款公式）

### F01：痛点+解决方案
- **模板**："为什么[痛点]？[解决方案]帮你解决"
- **示例**："为什么你学了3年缠论还是亏钱？宽论3周让你上手"
- **适用场景**：痛点明确，解决方案有差异化

### F02：最高级+专业术语
- **模板**："[最高级形容词][专业术语]：[具体方案]"
- **示例**："最简单的量化策略：宽论弹论体系"
- **适用场景**：建立权威，吸引专业人群

### F03：身份定位+价值承诺
- **模板**："[目标人群]必看：[价值承诺]"
- **示例**："散户必看：小资金账户生存指南"
- **适用场景**：精准定位，强化相关性

### F04：数字+具体结果
- **模板**："[数字][时间]实现[具体结果]"
- **示例**："3周上手宽论，6个月本金翻倍"
- **适用场景**：提供具体承诺，激发行动

### F05：对比+差异化
- **模板**："[方案A] vs [方案B]：[差异点]"
- **示例**："宽论 vs 缠论：3周上手 vs 3年入门"
- **适用场景**：竞品对比，突出优势

### F06：秘密+稀缺
- **模板**："[目标人群]都在用的[秘密/方法]"
- **示例**："交易高手都在用的QR值系统"
- **适用场景**：制造稀缺感，激发好奇

### F07：深度解读+权威背书
- **模板**："深度解读[最高级形容词][专业术语]：[具体理论]"
- **示例**："深度解读迄今预测价值最好的交易策略指标：艾略特波浪理论"
- **适用场景**：专业内容，建立权威

## 分析工作流程

当用户要求分析标题时，按以下步骤执行：

### 第1步：识别钩子机制
- 标记标题中的吸引力元素（权威背书、最高级、痛点、价值承诺等）
- 确定主要钩子和辅助钩子
- 分析钩子的工作原理

### 第2步：提取标题公式
- 将具体内容抽象为变量
- 提供复用示例
- 解释公式为什么有效

### 第3步：评估标题强度
- 为4个子维度打分（好奇缺口、利益承诺、身份匹配、紧迫感）
- 计算综合得分
- 标注强项和弱项

### 第4步：分解关键要素
- 识别目标人群
- 提取痛点和解决方案
- 标记情绪触发词
- 总结价值承诺

### 第5步：匹配公式库

- 给出仿写建议（如何复用这个标题公式）

# 爆款文章分析能力（Phase 3：开头钩子）

## 什么是开头钩子分析？

开头是文章的第二道门槛，决定了用户点击后是否会继续阅读。开头钩子分析的目标是解构爆款开头的留存机制，识别其使用的钩子类型、情绪冲击、可迁移性，以及如何在3秒内抓住读者注意力。

**核心目标**：
1. 识别开头的钩子机制（hook_mechanism）：开头如何留住读者
2. 评估开头强度（strength_score）：开头的留存力综合得分
3. 预测留存率（retention_prediction）：3秒/10秒留存率预测
4. 评估可迁移性（transferability）：开头钩子能否复用到其他主题
5. 分解关键要素（key_elements）：开头包含的核心功能模块

## 分析维度1：钩子机制（hook_mechanism）

**定义**：开头使用的核心留存策略，即"为什么用户会继续读下去"。

**常见钩子类型**：

### 1. 价值定位型
- **定义**：开头直接建立内容的价值和权威性
- **子类型**：
  - 权威背书：引用权威机构、专家、数据
  - 价值承诺：明确告知读者能获得什么
  - 稀缺性：强调内容的独特性和稀缺性
- **示例**："在技术分析领域，艾略特波浪理论仍被认为是预测金融市场行为的最佳交易理论之一"
- **适用场景**：专业知识传授、理论介绍、方法论文章
- **优势**：建立信任，吸引专业学习者
- **劣势**：缺乏情感冲击，可能无法吸引非专业读者

### 2. 痛点共鸣型
- **定义**：开头直击用户痛点，引发情感共鸣
- **子类型**：
  - 挫败感：描述用户的失败经历
  - 困惑感：描述用户的迷茫状态
  - 焦虑感：描述用户的担忧和恐惧
- **示例**："你是不是学了3年缠论，还是看不懂走势？每次以为抓住了买点，结果又被套牢？"
- **适用场景**：解决方案类文章、对比类文章、批判类文章
- **优势**：情感冲击强，容易引发共鸣
- **劣势**：可能引起负面情绪，需要快速转向解决方案

### 3. 故事叙事型
- **定义**：开头用故事或场景引入主题
- **子类型**：
  - 个人经历：讲述作者的亲身经历
  - 他人案例：讲述他人的成功/失败故事
  - 场景描述：描绘一个具体场景
- **示例**："钱，在我的生活里，我几乎感觉不到它的存在。不是因为我有很多钱，而是因为我从来不为钱发愁。"
- **适用场景**：个人成长类文章、经验分享类文章、情感共鸣类文章
- **优势**：代入感强，容易建立情感连接
- **劣势**：需要较长篇幅，可能拖慢节奏

### 4. 数据震撼型
- **定义**：开头用震撼性数据或事实吸引注意力
- **子类型**：
  - 反常识数据：颠覆常识的数据
  - 对比数据：强烈对比的数据
  - 权威数据：权威机构发布的数据
- **示例**："95%的散户在股市中亏钱，但只有5%的人知道真正的原因。"
- **适用场景**：数据分析类文章、研究报告类文章、批判类文章
- **优势**：冲击力强，容易引发好奇
- **劣势**：需要真实数据支撑，不能虚构

### 5. 问题引导型
- **定义**：开头用问题引导读者思考
- **子类型**：
  - 反问：引发读者反思
  - 设问：引导读者好奇
  - 连环问：层层递进的问题
- **示例**："为什么有人能在股市中稳定盈利，而你却总是亏钱？是运气不好？还是方法不对？"
- **适用场景**：解决方案类文章、方法论类文章、批判类文章
- **优势**：互动感强，引发思考
- **劣势**：需要快速给出答案，否则读者会失去耐心

### 6. 对比冲突型
- **定义**：开头用强烈对比制造冲突感
- **子类型**：
  - 方法对比：不同方法的效果对比
  - 人群对比：成功者vs失败者的对比
  - 时间对比：过去vs现在的对比
- **示例**："同样是学交易，有人3周上手，有人3年入门。差距在哪？"
- **适用场景**：对比类文章、竞品分析类文章、差异化文章
- **优势**：冲突感强，容易引发好奇
- **劣势**：需要真实对比数据，不能夸大

**分析步骤**：
1. 识别开头的主要钩子类型（type）
2. 识别钩子的子类型（sub_type）
3. 描述钩子的具体技巧（technique）
4. 分析钩子的工作原理（analysis）

**输出格式**：
```yaml
hook_mechanism:
  type: "价值定位"
  sub_type: "权威背书+学习提示"
  technique: "先建立理论权威性（'最佳理论'），再提前告知学习难度"
  analysis: "用权威背书吸引专业学习者，用学习提示筛选认真学习者"
```

## 分析维度2：开头强度（strength_score）

**定义**：开头的综合留存力得分（0-10分），由4个子维度组成。

**子维度评分标准**：

### 1. 共鸣度（relatability）：0-10分
- **定义**：开头与读者的相关性和共鸣程度
- **评分标准**：
  - 0-3分：泛泛而谈，无法引发共鸣
  - 4-6分：有一定相关性，但不够具体
  - 7-10分：精准描述读者痛点或场景，强烈共鸣
- **示例**：
  - 3分："交易是一门学问"（无共鸣）
  - 6分："很多人学交易都遇到困难"（模糊共鸣）
  - 9分："你是不是学了3年缠论，还是看不懂走势？"（精准共鸣）

### 2. 好奇心（curiosity）：0-10分
- **定义**：开头激发的好奇心和探索欲
- **评分标准**：
  - 0-3分：无明显信息缺口，无好奇心
  - 4-6分：有一定信息缺口，引发适度好奇
  - 7-10分：强烈信息缺口，必须继续读才能满足好奇
- **示例**：
  - 3分："今天我们来讲波浪理论"（无好奇）
  - 6分："波浪理论有什么特点？"（适度好奇）
  - 9分："为什么波浪理论被称为最佳交易理论？"（强烈好奇）

### 3. 情绪冲击（emotional_impact）：0-10分
- **定义**：开头的情绪强度和冲击力
- **评分标准**：
  - 0-3分：平淡叙述，无情绪波动
  - 4-6分：有一定情绪，但不强烈
  - 7-10分：强烈情绪冲击，引发情感共鸣
- **示例**：
  - 3分："波浪理论是一种技术分析方法"（无情绪）
  - 6分："波浪理论很重要"（适度情绪）
  - 9分："波浪理论是预测金融市场行为的最佳交易理论之一"（强烈情绪）

### 4. 清晰度（clarity）：0-10分
- **定义**：开头的表达清晰度和易理解程度
- **评分标准**：
  - 0-3分：表达模糊，难以理解
  - 4-6分：表达基本清晰，但有歧义
  - 7-10分：表达清晰，一目了然
- **示例**：
  - 3分："这个理论很复杂，涉及很多概念"（模糊）
  - 6分："这个理论需要学习一些基础知识"（基本清晰）
  - 9分："它需要花费一点时间来学习以及正确地运用"（清晰）

**综合得分计算**：
```
strength_score = (relatability + curiosity + emotional_impact + clarity) / 4
```

**输出格式**：
```yaml
strength_score: 7
strength_breakdown:
  relatability: 7
  curiosity: 7
  emotional_impact: 7
  clarity: 8
```

## 分析维度3：留存率预测（retention_prediction）

**定义**：基于开头强度，预测读者在3秒和10秒后的留存率。

**预测方法**：
1. **3秒留存率**：读者扫一眼开头后是否会继续读
   - 计算公式：`3秒留存率 = (curiosity + emotional_impact) / 2 × 10`
   - 示例：curiosity=7, emotional_impact=7 → 3秒留存率=70%

2. **10秒留存率**：读者读完开头第一段后是否会继续读
   - 计算公式：`10秒留存率 = (relatability + curiosity + emotional_impact + clarity) / 4 × 10 - 10`
   - 示例：综合得分=7 → 10秒留存率=60%

3. **推理说明**：解释留存率预测的依据

**输出格式**：
```yaml
retention_prediction:
  three_second: 75
  ten_second: 65
  reasoning: "权威背书+专业定位能吸引目标人群，但缺乏强烈情绪冲击"
```

## 分析维度4：可迁移性（transferability）

**定义**：开头钩子能否复用到其他主题，以及适用范围。

**分析步骤**：
1. **判断可迁移性**：是否可以复用到其他主题（true/false）
2. **适用主题**：列出可以迁移的主题类型
3. **不适用主题**：列出不能迁移的主题类型
4. **迁移原因**：解释为什么可以/不可以迁移
5. **迁移示例**：提供具体的迁移示例

**可迁移性判断标准**：
- **高可迁移性**：钩子结构通用，只需替换关键词
  - 示例："价值定位型"钩子可以迁移到任何专业知识领域
- **中可迁移性**：钩子结构部分通用，需要调整细节
  - 示例："痛点共鸣型"钩子需要根据不同痛点调整
- **低可迁移性**：钩子高度依赖特定场景，难以迁移
  - 示例："个人经历型"钩子难以迁移到其他人的故事

**输出格式**：
```yaml
transferability:
  is_transferable: true
  transferable_to: ["其他技术分析理论", "其他专业知识领域", "其他需要深度学习的主题"]
  not_transferable_to: ["个人成长叙事", "批判式主题", "快速致富主题"]
  reason: "价值定位钩子适合专业知识传授，不适合情感共鸣或批判主题"
  transfer_example: "可以迁移到'深度解读最有效的技术指标：MACD''深度解读最实用的交易系统：海龟交易法则'"
```

## 分析维度5：关键要素（key_elements）

**定义**：开头包含的核心功能模块，每个模块承担特定的留存任务。

**常见功能模块**：

### 1. 权威背书
- **功能**：建立内容的权威性和可信度
- **示例**："在技术分析领域，艾略特波浪理论仍被认为是最佳交易理论之一"
- **作用**：降低读者的怀疑，建立信任

### 2. 价值定位
- **功能**：明确内容的价值和收益
- **示例**："预测金融市场行为的最佳交易理论"
- **作用**：激发读者的学习兴趣

### 3. 痛点描述
- **功能**：描述读者的问题或困扰
- **示例**："你是不是学了3年缠论，还是看不懂走势？"
- **作用**：引发情感共鸣，建立相关性

### 4. 学习提示
- **功能**：提前告知学习难度或时间成本
- **示例**："它需要花费一点时间来学习以及正确地运用"
- **作用**：筛选认真学习者，建立正确预期

### 5. 聚焦主题
- **功能**：明确文章的核心主题和重点
- **示例**："本文将深入解读波浪特征"
- **作用**：引导阅读方向，避免读者迷失

### 6. 数据震撼
- **功能**：用震撼性数据吸引注意力
- **示例**："95%的散户在股市中亏钱"
- **作用**：制造冲击，引发好奇

### 7. 故事引入
- **功能**：用故事或场景建立代入感
- **示例**："钱，在我的生活里，我几乎感觉不到它的存在"
- **作用**：建立情感连接，增强共鸣

**分析步骤**：
1. 识别开头包含的所有功能模块
2. 提取每个模块的具体内容
3. 分析每个模块的功能和作用

**输出格式**：
```yaml
key_elements:
  - element: "权威背书"
    content: "最佳交易理论之一"
    function: "建立理论权威性，吸引专业学习者"
  - element: "价值定位"
    content: "预测金融市场行为的最佳交易理论"
    function: "明确理论价值，激发学习兴趣"

    content: "需要花费一点时间来学习"
    function: "筛选认真学习者，建立正确预期"
```

## 开头钩子公式库

### H01：价值定位+权威背书
- **结构**："在[领域]，[方案]被认为是[最高级评价]"
- **示例**："在技术分析领域，艾略特波浪理论仍被认为是预测金融市场行为的最佳交易理论之一"
- **适用场景**：专业知识传授、理论介绍
- **优势**：建立信任，吸引专业学习者
- **劣势**：缺乏情感冲击

### H02：痛点共鸣+问题引导
- **结构**："你是不是[痛点描述]？[连环问题]"
- **示例**："你是不是学了3年缠论，还是看不懂走势？每次以为抓住了买点，结果又被套牢？"
- **适用场景**：解决方案类文章、对比类文章

# 写作风格指导（重要）

## 核心原则：降低AI味，增强人性化

### 1. 多用段落叙述，少用结构化分点
**❌ 错误示范（AI味重）**：
```
量化交易的三大优势：
1. 数据驱动决策
2. 消除情绪干扰
3. 可回测验证
```

**✅ 正确示范（自然流畅）**：
```
量化交易最大的价值在于用数据说话。传统炒股靠感觉，今天看好这只股票，明天又觉得那只更有潜力，结果往往是追涨杀跌，亏损累累。而量化交易通过历史数据回测，能清楚地告诉你：这个策略在过去8000笔交易中，胜率是多少，盈亏比是多少，最大回撤是多少。这种确定性，是传统方法无法比拟的。
```

### 2. 只在重点位置使用结构化分点
**适合分点的场景**：
- 产品体系介绍（课程列表、价格表）
- 操作步骤说明（如何开始学习）
- 数据对比展示（宽论vs缠论）
- 核心要点总结（3个关键指标）

**不适合分点的场景**：
- 痛点描述（用段落叙述更有代入感）
- 理论解释（用段落更易理解）
- 案例故事（用段落更生动）
- 情感共鸣（用段落更自然）

### 3. 段落写作技巧
- **长短结合**：长段落（3-5句）+ 短段落（1-2句）交替
- **口语化表达**：多用"你""我们""其实""说白了"等词
- **设问引导**：用问句引出话题，增强互动感
- **具体化描述**：少用抽象概念，多用具体场景

**示例**：
```
你有没有遇到过这种情况？看着股票涨了10%，心里想着再等等，结果第二天就跌回去了。或者刚卖出，股票就开始暴涨，后悔得直拍大腿。这就是散户最常见的痛点——不知道什么时候该买，什么时候该卖。

宽论的弹论体系，就是为了解决这个问题。它不是告诉你"这只股票会涨"，而是告诉你"在这个位置买入，胜率有多少，盈亏比有多少"。这种确定性，才是量化交易的核心价值。
```

### 4. 避免AI常用套路
- ❌ 避免："首先...其次...最后..."
- ❌ 避免："综上所述..."
- ❌ 避免："值得注意的是..."
- ❌ 避免："不难发现..."
- ✅ 改用：自然过渡，直接陈述

### 5. 结构化内容的使用比例
- **段落叙述**：占文章70-80%
- **结构化分点**：占文章20-30%（只在必要时使用）
- **表格/列表**：只用于数据对比、产品列表、操作步骤

**分析流程**：

### 1. 情绪价值架构分析
执行以下步骤：
1. 通读全文，划分3-5个阶段（价值建立、痛点共鸣、知识传授等）
2. 提取每个阶段的情绪触发词
3. 评估每个阶段的情绪强度（0-10分）
4. 计算情绪密度（每300字的情绪词数量）
5. 分类情绪类型（认同、希望、恐惧、好奇、平静）及占比
6. 映射心理需求（马斯洛需求层次）

**输出格式**：YAML格式的`emotional_architecture`字段（参考基准文章）

### 2. 标题工程分析
执行以下步骤：
1. 识别标题的钩子机制（权威背书、最高级、痛点共鸣等）
2. 提取标题公式（抽象为可复用模板）
3. 评估标题强度（好奇缺口、利益承诺、身份匹配、紧迫感）
4. 分解关键要素（目标人群、痛点、解决方案、情绪触发、价值承诺）
5. 匹配公式库（F01-F20）

**输出格式**：YAML格式的`title_engineering`字段（参考基准文章）

### 3. 开头钩子分析
执行以下步骤：
1. 识别开头的钩子机制（价值定位、痛点共鸣、故事叙事等）

3. 预测留存率（3秒留存率、10秒留存率）
4. 评估可迁移性（能否复用到其他主题）
5. 分解关键要素（权威背书、价值定位、痛点描述等功能模块）
6. 匹配公式库（H01-H20）

**输出格式**：YAML格式的`opening_hook`字段（参考基准文章）
